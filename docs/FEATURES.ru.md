# Возможности

## Во время встречи

- **Живая стенограмма** — реплики склеиваются в абзацы по говорящим, а не
  сыплются строками-чанками. Лёгкая модель дорисовывает границы реплик
  внутри абзаца («— …»), не меняя ни слова (строгая валидация).
- **Живая диаризация** — «Собеседник 1/2/…» по голосам с первых секунд.
  Имя владельца не угадывается никогда; ваш микрофон подписан вашим именем
  из конфига.
- **Живые имена** — как только человек представился (или к нему обратились
  и он ответил), метка «Собеседник N» заменяется на имя — задним числом по
  всей стенограмме и во всех будущих репликах. Модель обязана найти имя в
  тексте — галлюцинации отсекаются.
- **Мгновенный ответ (⚡)** — вопрос собеседника детектится по пунктуации
  STT и стартовым словам; готовый ответ от первого лица приходит за ~2-3 с.
  Над ответом показан сам вопрос.
- **Облачный ответ (☁️, опция)** — параллельно локальному тот же вопрос
  уходит Claude (ваша подписка, `claude` CLI): локальный ответ мгновенный,
  облачный — глубже на 10-20 с позже. По умолчанию выключено.
- **Автотезисы** — 📌 факты/решения, 💎 важное, 💭 идеи по ходу разговора;
  тяжёлая модель периодически пересматривает заметки лёгкой (🔬).
- **Дежавю (⏮)** — если тема разговора совпала с «Ядром» графа (сквозной
  темой прошлых встреч), в тезисы приходит «⏮ уже обсуждалось 15.07,
  статус: …». Матч по основам слов, работает с русскими падежами.
- **Живой черновик минуток** — обновляется каждые ~2.5 минуты; финальный
  протокол — по команде или после стопа.
- **Вопрос ассистенту** — спросите текстом посреди встречи: ответ строится
  сначала по живой стенограмме, потом по памяти графа, потом по знаниям
  модели (с пометкой источника).

## После встречи (автоматически)

1. **Пересборка стенограммы** — полная запись диаризуется заново по
   каналам: фильтр эха, слияние микрофрагментов, чистые абзацы, имена.
2. **Обновление графа** — сущности (Люди/Системы), решения, поручения,
   «Ядра» со статусом и хроникой; заметка встречи со связями.
3. **Архив встречи** — папка «дата — название (2-3 слова)» со всеми
   документами и ссылкой, открывающей граф в Obsidian.
4. **Саммари** — выжимка на минуту чтения: суть одной строкой → темы →
   решили → поручения (кто/что/срок) → открытые вопросы → **связь с
   прошлыми встречами** («было: … (15.07) → сегодня: …») → навигация
   к Минуткам/Разбору/Стенограмме.
5. **Разбор** (опция) — вопросы и ответы встречи, задачи, варианты решений
   открытых вопросов, рекомендации к следующей встрече.

## Вне встреч

- **Диктовка** (глобальный хоткей) — наговорили → распознано локально →
  вставлено в активное поле; буфер обмена восстанавливается, включая
  картинки.
- **Голосовая заметка** — наговорили мысль → модель причёсывает текст,
  ставит заголовок, извлекает задачи → файл в графе (`Заметки/`) +
  заметка запоминается в памяти. Сырой текст сохраняется рядом
  («Как сказано»).

## Формат документов

Все документы — plain-markdown, читаются без рендера: списки с жирным
ключом вместо таблиц, короткие блоки, одинаковая структура каждый раз,
главное — первой строкой (BLUF). Слои: Саммари (минута) → Минутки →
Разбор → Стенограмма.
